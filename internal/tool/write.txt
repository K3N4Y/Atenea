Crea un archivo NUEVO en el workspace con el contenido exacto que le pasas.

Uso:
- path es la ruta del archivo relativa al workspace; content es el contenido completo. Los directorios padre que falten se crean solos.
- Solo crea archivos nuevos. Si el archivo YA existe, write falla: para modificar uno existente usa read + edit (edit ancla contra el contenido que leiste). No uses write para sobrescribir.
- Tras escribir devuelve el header [path#HASH] del contenido nuevo: puedes encadenar un edit a ese header sin re-leer el archivo (write -> edit).
- Prefiere modificar archivos existentes antes que crear nuevos; crea un archivo solo cuando hace falta de verdad.
- Los archivos se escriben con saltos de linea LF.
